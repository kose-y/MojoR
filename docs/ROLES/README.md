# Role-Based Onboarding Guides

Use these guides for a focused first 2 hours based on your role.

## Available Guides

- [`USER_FIRST_2_HOURS.md`](./USER_FIRST_2_HOURS.md)
 - For package users who want to run kernels quickly.
- [`COMPILER_DEV_FIRST_2_HOURS.md`](./COMPILER_DEV_FIRST_2_HOURS.md)
 - For contributors working on AST/IR/transpilation passes.
- [`BACKEND_RUNTIME_FIRST_2_HOURS.md`](./BACKEND_RUNTIME_FIRST_2_HOURS.md)
 - For contributors working on Mojo backend, bridge, and ABI/runtime integration.
- [`RELEASE_MAINTAINER_FIRST_2_HOURS.md`](./RELEASE_MAINTAINER_FIRST_2_HOURS.md)
 - For package maintainers preparing docs/build/test/release flows.

## Suggested Reading Order

1. Start with the guide that matches your primary role.
2. Read [`docs/GLOSSARY.md`](../GLOSSARY.md) for plain-language terminology.
3. If you are contributing code, read the Compiler Dev or Backend Runtime guide next.
4. Before packaging/release, run through the Release Maintainer guide.
